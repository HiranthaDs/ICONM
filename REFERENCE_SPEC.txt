INVOICE SAVE TRANSACTION LOGIC

When saving retail invoice:

BEGIN TRANSACTION

1. Validate customer fields.
2. Validate cart not empty.
3. Validate stock availability.
4. Validate each serialized unit is Available.
5. Create or update customer.
6. Create invoice.
7. Create invoice items.
8. Create payment rows.
9. For each serialized item:
   update stock_units set status='Sold', invoice_id=invoice.id, sold_at=now.
10. For each bulk item:
    reduce products.available_quantity.
11. Update products sold_quantity.
12. Generate PDF.
13. Save pdf_path in invoice.
    COMMIT

If any step fails:
ROLLBACK and show clear error.

WHOLESALE ISSUE TRANSACTION

BEGIN TRANSACTION

1. Validate partner/shop.
2. Validate issue cart.
3. Validate stock.
4. Create partner_invoice.
5. Save partner_invoice_items.
6. Save initial payment rows if paid amount entered.
7. Mark serialized units as PartnerIssued.
8. Reduce bulk quantity.
9. Generate wholesale invoice PDF.
10. Save PDF path.
    COMMIT

PARTNER PAYMENT TRANSACTION

1. Validate shop selected.

4. Allocate payment FIFO.
